    Runtime Enhancements

    Lambda Expression

    Lambda 也是 C++ 中很重要的特性，相信大家都不陌生

    [capture list] (parameter list) mutable(optional) exception attribute -> return type {
// function body
}

    其中 [capture list] 的用途是抓取目前 scope 的變數至 lambda expression 中，有以下幾種用法

    • []: 完全不使用外部變數，
    • [=]: capture by value
    • [&]: capture by reference
    • [x, &y]: capture x by value, y by reference

    Generic Lambda

    上一篇有說過函式的參數不能用 auto 類型，是因為會和 template 產生衝突，但是由於 Lambda 並不是普通函式所以也不能使用 template，這樣導致我們沒辦法寫出 Generic Lambda

    在 C++14 後改善了這項問題，我們可以用 auto 當作 Lambda 的參數類型

    auto generic = [] (auto a, auto b) {
    return x + y;
};
cout << generic(1, 2) << endl;
cout << generic(1.1, 1.2) << endl;

    Function Object Wrapper

    std::function

    由於 C++ 的許多特性讓許多本質上是函式的物件型態不一樣

    1. 因為 lambda expression 不是函式，所以 function pointer 不能指向 lambda expression
    2. static member function 跟 non-static member function 型態不同，印為non-static member function會綁定於一個物件
    3. functor: 可以當作函式呼叫的物件，最簡單的做法就是定義一個 class 然後在其中定義 operator()

    因此 C++11 引入了 function，function 可以用來存任何可以被呼叫的物件，也就是一個函式的 container

    #include <functional>
#include <iostream>

int foo(int para) {
    return para;
}

int main() {
    std::function<int(int)> func = foo;

    int important = 10;
    std::function<int(int)> func2 = [&](int value) -> int {
        return 1+value+important;
    };
    std::cout << func(10) << std::endl;
    std::cout << func2(10) << std::endl;
}

    std::bind && std::placeholder

    bind 也是一種 function wrapper，直接看例子

    int add(int a, int b){
    return a + b;
}

    假設今天你只需要 a + 1，這時你不用重寫一個函式

    auto add1 = bind(add, placeholders::_1, 1);
add1(3); // 4

    placeholders::_1 意思就是將你後來給的第一個參數傳給 add，如此一來就是 3 + 1，使用的場景就是先寫好一個通用的函式，再透過 bind 組成各個符合需求的函式如 add1(int a)

    rvalue Reference

    先看看以下四種定義

    1. lvalue: 在一個表達式執行結束後會持續存在的物件，通常是賦值表達式左側的變數
    2. rvalue: 表達式執行結束後會馬上消失的物件，通常是指等號右側的值
    3. pvalue: pure rvalue 指各種常量和 anonymous temporary object
    4. xvalue: expiring value 即將被摧毀但是可以被移動的值

    我們用以下例子來了解 xvalue

    std::vector<int> foo() {
    std::vector<int> temp = {1, 2, 3, 4};
    return temp;
}

std::vector<int> v = foo();

    當 foo() 被呼叫時，它會先建立一個 temp 變數回傳後再複製給 v，最後刪除 temp，如果此時 temp 非常大的話整個過程會非常耗時( v 是 lvalue 而 foo()回傳的值是 pvalue )，在最後一行賦值的過程中，foo()的回傳值是一個暫時值，在這句表達式結束後就會消失，xvalue 定義這種暫時值可以被識別和移動。

    為了取得 xvalue，我們需要透過 rvalue reference: type &&，這讓 xvalue 的生命得以延續，只要變數還存在xvalue就不會被刪除

    std::move 可以將 lvalue 轉為 rvalue

    string lv1 = "string,";
string &&rv1 = move(lv1);
cout << rv1 << endl; // string,

string &&rv2 = lv1 + lv1;
rv2 += "string"
cout << rv2 << endl; // string,string,string
